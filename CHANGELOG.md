## How to Update

When using this template:

1. Update this CHANGELOG.md with your project changes
2. Follow the format above
3. Keep version numbers synchronized with your releases

## Version Guidelines

- **Major** (1.0.0): Significant changes or breaking updates
- **Minor** (0.1.0): New features, non-breaking
- **Patch** (0.0.1): Bug fixes, documentation updates
